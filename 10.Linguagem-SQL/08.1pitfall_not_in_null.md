# Pitfall com NOT IN e NULL no SQL


# O problema

Em SQL, `NULL` representa ausência de valor.

Ele não significa zero, texto vazio ou falso.

`NULL` significa:

```text
valor desconhecido
```

Por isso, comparações com `NULL` podem gerar resultados inesperados.

Exemplo:

```sql
SELECT 1 = NULL;
SELECT 1 = 1;
SELECT 1 = 2;
```

O resultado não é verdadeiro nem falso.

O resultado é desconhecido.

---

#  Criando as tabelas de exemplo

Vamos criar duas tabelas simples:

- `cliente2`
- `pedido2`

A tabela `cliente2` representa clientes.

A tabela `pedido2` representa pedidos.

Diferente do modelo principal, aqui vamos permitir que `pedido2.cliente_id` aceite `NULL`, justamente para demonstrar o problema.

---

## Criar tabela cliente2

```sql
DROP TABLE IF EXISTS pedido2;
DROP TABLE IF EXISTS cliente2;

CREATE TABLE cliente2 (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nome VARCHAR(100) NOT NULL
);
```

---

## Criar tabela pedido2

```sql
CREATE TABLE pedido2 (
    id INT AUTO_INCREMENT PRIMARY KEY,
    cliente_id INT NULL,
    data_pedido DATE NOT NULL
);
```

Repare que a coluna `cliente_id` permite `NULL`:

```sql
cliente_id INT NULL
```

Isso foi feito de propósito para o exemplo.

---

# Inserindo dados de exemplo

##  Inserir clientes

```sql
INSERT INTO cliente2 (nome) VALUES
('Ana'),
('João'),
('Maria'),
('Carlos');
```

Agora temos quatro clientes:

```text
1 - Ana
2 - João
3 - Maria
4 - Carlos
```

---

##  Inserir pedidos

```sql
INSERT INTO pedido2 (cliente_id, data_pedido) VALUES
(1, '2024-01-10'),
(3, '2024-01-15'),
(NULL, '2024-01-20');
```

Agora temos três pedidos:

```text
Pedido 1 pertence à Ana
Pedido 2 pertence à Maria
Pedido 3 está com cliente_id NULL
```

Esse terceiro pedido pode representar, por exemplo:

- erro de carga;
- dado incompleto;
- pedido órfão;
- integração com problema;
- registro ainda não associado a um cliente.

---

# Conferindo os dados

##  Consultar clientes

```sql
SELECT *
FROM cliente2;
```

Resultado esperado:

```text
+----+--------+
| id | nome   |
+----+--------+
|  1 | Ana    |
|  2 | João   |
|  3 | Maria  |
|  4 | Carlos |
+----+--------+
```

---

##  Consultar pedidos

```sql
SELECT *
FROM pedido2;
```

Resultado esperado:

```text
+----+------------+-------------+
| id | cliente_id | data_pedido |
+----+------------+-------------+
|  1 |          1 | 2024-01-10  |
|  2 |          3 | 2024-01-15  |
|  3 |       NULL | 2024-01-20  |
+----+------------+-------------+
```

---

# O objetivo da consulta

Queremos responder à seguinte pergunta:

> Quais clientes nunca fizeram pedido?

Olhando os dados manualmente:

- Ana fez pedido.
- Maria fez pedido.
- João não fez pedido.
- Carlos não fez pedido.

Então o resultado esperado seria:

```text
João
Carlos
```

---

#  Tentativa usando NOT IN

Uma primeira tentativa comum seria usar `NOT IN`.

```sql
SELECT
    c.id,
    c.nome
FROM cliente2 c
WHERE c.id NOT IN (
    SELECT p.cliente_id
    FROM pedido2 p
);
```

Parece correto, mas pode dar problema.

---

#  Entendendo a subquery

Vamos executar apenas a subquery:

```sql
SELECT p.cliente_id
FROM pedido2 p;
```

Resultado:

```text
+------------+
| cliente_id |
+------------+
|          1 |
|          3 |
|       NULL |
+------------+
```

Ou seja, a subquery retorna:

```text
1, 3, NULL
```

Então a consulta com `NOT IN` vira, na prática:

```sql
SELECT
    c.id,
    c.nome
FROM cliente2 c
WHERE c.id NOT IN (1, 3, NULL);
```

---

#  Por que isso dá problema?

Vamos analisar o cliente João.

João tem `id = 2`.

A condição seria:

```sql
2 NOT IN (1, 3, NULL)
```

Isso é equivalente a perguntar:

```sql
2 <> 1
AND 2 <> 3
AND 2 <> NULL
```

Agora vamos avaliar cada parte:

```text
2 <> 1     => verdadeiro
2 <> 3     => verdadeiro
2 <> NULL  => desconhecido
```

O problema está aqui:

```sql
2 <> NULL
```

Como `NULL` é desconhecido, o SQL não consegue afirmar que `2` é diferente de `NULL`.

O resultado final fica:

```text
verdadeiro AND verdadeiro AND desconhecido
```

Em SQL, isso não é tratado como verdadeiro.

Então o cliente João não aparece no resultado.

O mesmo acontece com Carlos.

---

# Resultado inesperado

A consulta abaixo pode não retornar nenhum cliente:

```sql
SELECT
    c.id,
    c.nome
FROM cliente2 c
WHERE c.id NOT IN (
    SELECT p.cliente_id
    FROM pedido2 p
);
```

Mesmo João e Carlos não tendo pedidos, eles podem não aparecer por causa do `NULL` retornado pela subquery.

---

# Como corrigir usando NOT IN

Uma forma de corrigir é remover o `NULL` da subquery.

```sql
SELECT
    c.id,
    c.nome
FROM cliente2 c
WHERE c.id NOT IN (
    SELECT p.cliente_id
    FROM pedido2 p
    WHERE p.cliente_id IS NOT NULL
);
```

Agora a subquery retorna apenas:

```text
1, 3
```

Então a consulta fica:

```sql
WHERE c.id NOT IN (1, 3)
```

Resultado esperado:

```text
+----+--------+
| id | nome   |
+----+--------+
|  2 | João   |
|  4 | Carlos |
+----+--------+
```

---

#  Melhor solução: NOT EXISTS

A forma mais segura e recomendada para esse tipo de consulta é usar `NOT EXISTS`.

```sql
SELECT
    c.id,
    c.nome
FROM cliente2 c
WHERE NOT EXISTS (
    SELECT 1
    FROM pedido2 p
    WHERE p.cliente_id = c.id
);
```

---

# Por que NOT EXISTS funciona melhor?

O `NOT EXISTS` não cria uma lista com valores como:

```text
1, 3, NULL
```

Em vez disso, ele faz uma pergunta para cada cliente:

```text
Existe algum pedido para este cliente?
```

Para Ana:

```text
Existe pedido com cliente_id = 1?
Sim.
Então Ana não entra no resultado.
```

Para João:

```text
Existe pedido com cliente_id = 2?
Não.
Então João entra no resultado.
```

Para Maria:

```text
Existe pedido com cliente_id = 3?
Sim.
Então Maria não entra no resultado.
```

Para Carlos:

```text
Existe pedido com cliente_id = 4?
Não.
Então Carlos entra no resultado.
```

Resultado:

```text
João
Carlos
```

---

#  Comparando as abordagens

##  NOT IN com risco

```sql
SELECT
    c.id,
    c.nome
FROM cliente2 c
WHERE c.id NOT IN (
    SELECT p.cliente_id
    FROM pedido2 p
);
```

Problema:

```text
Se a subquery retornar NULL, o resultado pode ser inesperado.
```

---

## NOT IN corrigido

```sql
SELECT
    c.id,
    c.nome
FROM cliente2 c
WHERE c.id NOT IN (
    SELECT p.cliente_id
    FROM pedido2 p
    WHERE p.cliente_id IS NOT NULL
);
```

Melhor, porque remove o `NULL`.

---

##  NOT EXISTS

```sql
SELECT
    c.id,
    c.nome
FROM cliente2 c
WHERE NOT EXISTS (
    SELECT 1
    FROM pedido2 p
    WHERE p.cliente_id = c.id
);
```

Mais seguro e geralmente mais recomendado para buscar ausência de relacionamento.
